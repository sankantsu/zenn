## やりたいこと

Github Copilot CLI を使っていて、 UI 面でスクロールしづらいのがやや不満でした。
マウス (タッチパッド) ではスクロールできますが、スクロール量が制御しづらく感じます。

キーボード操作では、私が知る限りでは Home/End で buffer 先頭/末尾や PageUp/PageDown でのページ単位でしかスクロールできません。
もっと細かく、行単位でのスクロールがしたいです。
https://github.com/github/copilot-cli/issues/2750

## 解決策

マウスではスクロールできるんだから、マウス操作相当の入力を keymap で送ってしまえばいいのではないかと考えました。

Wezterm では、以下のように実現できました。
`Ctrl + Shift + ↑/↓` で mouse 操作相当の control sequence を送って行単位でスクロールができるようになります。

```lua
local wezterm = require 'wezterm'
local config = wezterm.config_builder()
local act = wezterm.action

config.keys = {
  {
    key = 'UpArrow',
    mods = 'CTRL|SHIFT',
    action = act.SendString('\x1b[<64;1;1M'),
  },
  {
    key = 'DownArrow',
    mods = 'CTRL|SHIFT',
    action = act.SendString('\x1b[<65;1;1M'),
  },
}

-- and finally, return the configuration to wezterm
return config
```

おそらく、他の端末でもキーマップをある程度カスタマイズ性があるものであれば可能でしょう。
